# UIデザイン最適化提案

## 現状の問題点

1. **メニューの重複**: 右上の歯車メニューとサイドバーの「設定」セクションに「プロフィール」が重複
2. **メニュー構造の不明確さ**: 設定項目が散在している
3. **ユーザビリティ**: 同じ機能へのアクセス方法が複数あると混乱を招く

## 最適化方針

### 1. 右上ヘッダーメニュー（ユーザーアカウント関連）

**役割**: ユーザーアカウントの基本操作に限定

**内容**:
- ユーザー名表示
- ログアウト

**変更**: プロフィールリンクを削除（サイドバーに統一）

### 2. サイドバー「設定」セクション（アプリケーション設定）

**役割**: アプリケーションの各種設定に集約

**構成**:
```
設定
├── プロフィール（ユーザー情報の編集）
├── AIスクリプト管理（管理者のみ）
├── マイスクリプト（ユーザー用）
├── クラウドストレージ設定
└── ストレージファイル一覧
```

### 3. メニュー構造の再編成

**推奨構造**:
```
Home
└── ダッシュボード

AI相談
└── AI相談センター

Management
├── 借入管理
├── 月次PL
├── 年次財務諸表
├── 株主情報
├── ローカルベンチマーク
└── ノート

データ取り込み
├── CSV取り込み
│   ├── 月次PL
│   └── 年次決算
└── OCR読み込み

設定
├── プロフィール
├── AIスクリプト管理（管理者のみ）
├── マイスクリプト
├── クラウドストレージ設定
└── ストレージファイル一覧

Consultant Menu（コンサルタントのみ）
└── クライアント一覧

管理（スーパーユーザーのみ）
└── 管理画面
```

## 実装内容

### 1. ヘッダーメニューの簡素化

- プロフィールリンクを削除
- ユーザー名とログアウトのみ表示

### 2. サイドバー設定セクションの拡充

- クラウドストレージ設定を追加
- ストレージファイル一覧を追加
- アイコンを統一（⚙️）

### 3. 視覚的な改善

- 設定セクションを折りたたみ可能にする（オプション）
- アイコンの統一
- 階層構造の明確化
